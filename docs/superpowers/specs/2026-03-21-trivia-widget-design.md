# RiffTrax Trivia Widget — Design Spec

**Date:** 2026-03-21
**Status:** Approved

---

## Overview

Add a second Übersicht desktop widget that displays scrolling trivia about the current RiffTrax presentation. Trivia is generated by the Claude API (RiffTrax-flavored, 2–3 sentences) and fetched by a lightweight watcher script that monitors the existing now-playing file written by `bot.py`.

Nothing in the existing system changes.

---

## Architecture & Data Flow

```
bot.py (unchanged)
    └── writes ~/.rifftrax_now_playing.txt

trivia_watcher.py (new)
    ├── polls ~/.rifftrax_now_playing.txt every 5s
    ├── on title change → calls Claude API
    │       prompt: RiffTrax-flavored trivia, 2-3 sentences
    │       fallback: generic RiffTrax-flavored quip on API error
    ├── writes result to ~/.rifftrax_trivia.txt
    └── reads API key from ~/.rifftrax_anthropic_key (raw key, single line)

widget/rifftrax-trivia.widget/index.coffee (new)
    ├── polls ~/.rifftrax_trivia.txt every 5s
    ├── same black/red aesthetic as existing widget
    ├── positioned below existing widget (top: 160px, left: 40px)
    └── CSS marquee scroll on trivia text
```

The two widgets are independent. The trivia widget trails the now-playing widget by the API call duration (~1–3s), which is acceptable.

---

## Components

### `trivia_watcher.py`

- **Startup:** reads `~/.rifftrax_anthropic_key` (raw API key, single line). Exits with a clear error message if the file is missing.
- **Poll loop:** every 5 seconds, reads `~/.rifftrax_now_playing.txt` and compares to `last_title`.
- **On title change:** calls Claude API with a prompt requesting 2–3 sentences of RiffTrax-flavored trivia about the movie. Writes the result to `~/.rifftrax_trivia.txt`.
- **On title cleared (empty file):** writes an empty `~/.rifftrax_trivia.txt` so the widget hides.
- **On API error:** writes a fallback RiffTrax-flavored one-liner (e.g., "Even our robots couldn't find anything nice to say about this one.") and logs the error to stdout.
- **Runs until Ctrl+C**, same pattern as `bot.py`.

#### Claude API Prompt

```
You are a writer for RiffTrax, the comedy riffing show. Write 2-3 short, punchy sentences of trivia about "{title}" in the RiffTrax voice — dry wit, affectionate mockery, genuine facts wrapped in a joke. If you don't recognize the title, write something generically funny about low-budget cinema. No bullet points, no lists, just flowing sentences.
```

### `widget/rifftrax-trivia.widget/index.coffee`

- `command`: `cat ~/.rifftrax_trivia.txt 2>/dev/null`
- `refreshFrequency`: 5000
- **Position:** `top: 160px`, `left: 40px` (clears the now-playing widget above it)
- **Style:** same black background (`rgba(0,0,0,0.75)`), red left border (`#e50914`), same font and border-radius as existing widget
- **Scroll:** CSS `@keyframes marquee` — text translates from 100% to -100% over ~25s, loops infinitely. Fixed-width container with `overflow: hidden`.
- **Visibility:** hidden by default (`.visible` class added when output is non-empty), same pattern as existing widget.

---

## Error Handling

| Situation | Behavior |
|---|---|
| `~/.rifftrax_anthropic_key` missing | Script exits immediately with a descriptive error message |
| API call fails | Write fallback quip to trivia file; log error to stdout |
| Title file absent or empty | Widget stays hidden; watcher waits quietly, no API call |
| Title unchanged between polls | No API call, no file write |

## File Layout

```
trivia_watcher.py                          # new — run alongside bot.py
widget/
  rifftrax.widget/index.coffee             # unchanged
  rifftrax-trivia.widget/index.coffee      # new
~/.rifftrax_anthropic_key                  # user-created, raw API key, single line
~/.rifftrax_trivia.txt                     # written by watcher, read by widget
```

---

## Out of Scope

- No changes to `bot.py`
- No changes to the existing now-playing widget
- No configuration UI or settings file beyond the key file
- No retry logic on API failure (fallback quip is sufficient)
